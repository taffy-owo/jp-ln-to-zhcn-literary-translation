# 文化负载词注释策略

## 核心原则

轻小说翻译中遇到文化负载词时，默认策略是**不加脚注**。但"不加脚注"不等于"完全不解释"——需要根据词的类型和读者理解需要，在四种处理方式中选择最合适的。

## 四种处理方式

| 方式 | 说明 | 适用场景 | 示例 |
|---|---|---|---|
| **直译** | 直接翻译，不额外解释 | 中文读者无需解释即可理解 | 新年参拜 → 去神社参拜 |
| **文内轻解释** | 在不改变叙述节奏的前提下，用几个字嵌入说明 | 不解释会影响理解，但正式注释太重 | お盆 → 盂兰盆节假期（类似中元节） |
| **保留原样** | 保留日文概念或已被中文读者广泛接受的译法 | 二次元/ACG 读者熟知的术语，保留增加辨识度 | 异世界、转生、勇者、魔王 |
| **旁注** | 用括号或短注释做极简说明 | 极少使用；仅当概念对剧情理解至关重要且无法自然文内化时 | ※极少使用 |

## 五类分类处理规则

### 1. 设定词（世界观/系统）
- **默认**：保留原样或直译
- **规则**：已被 fandom 广泛接受的词（异世界、转生、勇者、公会、技能、等级）保留
- **注意**：作品独创的设定词，首次出现时可文内轻解释，之后直接使用
- **禁止**：不要把所有日文设定词都解释成普通汉语（会丢掉类型气味）

### 2. 学校/敬称/社会制度
- **默认**：直译或中文等效替换
- **规则**：
  - 学年制（高一/高二/高三）直接使用中文对应
  - 部活 → 社团活动
  - 文化祭 → 文化节/校园祭（视作品风格）
  - 修学旅行 → 修学旅行（已被中文读者接受）或 毕业旅行/学年旅行
  - 先輩/後輩 → 学长学姐/学弟学妹（校园），前辈/后辈（职场）
- **注意**：不要把日本特有但中文读者可以直觉理解的词过度解释

### 3. 双关与文字游戏
- **默认**：优先中文重写，保留喜剧效果
- **规则**：
  - 能用中文谐音/双关替代的，替代
  - 无法替代但笑点很重要的，保留效果但可以改变形式
  - 纯依赖日语语音的冷笑话，如果不影响剧情，可以适当弱化或改写成中文读者能笑的形式
- **禁止**：长译注解释笑话（笑话解释了就不好笑了）

### 4. 食物与日常生活
- **默认**：直译，中文读者大多已熟悉
- **规则**：
  - 常见日料（拉面、饭团、味噌汤、便当、章鱼烧、大福）直接使用
  - 不常见的地方特产或季节食物，首次出现时可文内轻解释
  - 不要为了"本土化"把日式食物改成中式食物
- **示例**：いなり寿司 → 豆皮寿司（首次可加"裹着甜豆皮的"）

### 5. 礼俗与社会习惯
- **默认**：文内轻解释或直译
- **规则**：
  - 初詣 → 新年参拜（多数读者理解）
  - お花見 → 赏樱（直译即可）
  - 七夕（日本的 7/7）→ 七夕（不需要解释与中国七夕的日期差异，除非剧情涉及）
  - 成人式 → 成人礼（直译）
  - 土下座 → 下跪道歉/五体投地（视严肃程度）
- **注意**：如果礼俗的具体规则对剧情理解很重要（比如角色不了解某个习俗而闹笑话），需要通过上下文让中文读者也能理解笑话的由来

## 决策流程

遇到文化负载词时，按以下顺序判断：

1. **中文读者能直接理解吗？** → 直译
2. **是 ACG/fandom 常用术语吗？** → 保留原样
3. **不解释会影响剧情理解吗？** → 文内轻解释
4. **以上都不行，且对剧情至关重要？** → 旁注（极少使用）
5. **无论如何都别做的事**：不要为了显示"翻译认真"而过度注释日常用语

## 项目级覆盖

每个项目的 `meta/style_guide.zh-CN.md` 可以覆盖本文件的默认策略。如果用户有特殊要求（如"保留所有日式称呼后缀""加注释版"），以用户要求为准。

## 与 glossary 的关系

文化负载词的处理决定应记录在 `meta/glossary.csv` 中：
- `category` 标记为 `culture_loaded`
- `cultural_note` 字段记录处理理由
- 首次出现时的文内解释不需要每次重复，后续章节直接使用已确定的译法
