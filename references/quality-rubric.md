# 质量标准

## MQM 风格维度
| 维度 | 权重 | 检查重点 | 常见错误 |
|---|---:|---|---|
| 准确性 | 30 | 事实、因果、指代、关系、伏笔 | 漏译、错译、反译、主语补错 |
| 流畅度 | 20 | 是否像中文小说 | 日语语序、僵硬、翻译腔、连接词堆叠 |
| 角色声音 | 15 | 说话人差异、礼貌层级、年龄感 | 所有人同声、敬语扁平化 |
| 术语/连续性 | 15 | 人名、称呼、世界观、口头禅 | 漂移、多套译法 |
| 笑点/文化 | 10 | 梗、双关、吐槽、文化词 | 笑点死掉、过度解释 |
| 节奏/可读性 | 10 | 段落推进、情绪弧线、转折密度 | 告白/创伤/冲突被写平，段首连接词重复 |

## 出厂门槛
- 人名/地名一致：100%。
- 关键称呼漂移：0 个重大问题。
- 高风险 GPT 腔误用：0 个未处理。
- 连接词/转折词过密：0 个明显模式化问题；同一段首连接词不得反复刷屏。
- 漏译/重复：0 个重大问题。
- 章节顺序/分块合并：0 个断裂。
- 中文标点和引号错误：接近 0。
- 角色声音：主要角色可区分。

## 失败条件（硬性阈值）

以下任一条件触发时，**不得宣称翻译完成**，必须先修复再交付：

| 条件 | 阈值 | 说明 |
|---|---|---|
| 事实错译 | ≥ 1 处 critical | 说话人归属错、因果反转、剧情事实改变 |
| 漏段 | ≥ 1 段 | 原文有但译文缺失的完整段落 |
| 术语漂移 | ≥ 2 处 | 同一术语/称呼在同一章内出现多种译法且无剧情原因 |
| 角色声线不可区分 | 主要角色 | 两个主要角色的台词读起来像同一个人说的 |
| 段首连接词连续重复 | ≥ 3 处未处理 | 相邻段落以"可是/不过/即便如此"等开头且审校未处理 |
| 章节拼接错乱 | ≥ 1 处 | 场景突跳、时间断裂、段落重复、段落缺失 |
| 未翻译日文残留 | ≥ 1 处 | 日文拟声拟态词、台词、句子残留在中文译文中 |
| 亲密/成人场景遮掩 | ≥ 1 处 | 原文有的感官细节被删节、弱化或委婉替换 |

## 混合评估质量卡

每章完成后，输出以下质量卡作为证据（不要空泛自夸）：

```markdown
# 章节质量卡 - [章节号]

## Hard Accuracy
- 事实错误：0 / N 处
- 说话人归属确认：全部确认 / X 处存疑
- 关键因果/伏笔：保留 / 遗漏

## Voice
- 主要角色声线稳定性：稳定 / 存在 X 处漂移
- 称呼一致性：一致 / X 处变化（有剧情原因 / 无剧情原因）

## Continuity
- 与前章衔接：无断裂 / X 处需检查
- 术语一致：一致 / X 处漂移
- 口头禅/反复句：保持 / 遗失

## Naturalness
- 连接词密度：正常 / 偏高（X 个段首重复）
- GPT 腔命中：0 / X 处（已处理 / 未处理）
- 翻译腔残留：0 / X 处

## Open Issues
- [列出未决问题]

## Golden Set 回归（如适用）
- 对照样张：[编号]
- 偏离点：[列出]
- 判定：进步 / 等价 / 退化
```

## 评分解释
- 90+：可进入人工轻校。
- 80-89：整体强，但需统一文风和术语。
- 70-79：需要重点修复。
- 70 以下：问题章节重翻，不要局部补丁。

## 审校层级
1. Smoke test：文件数、编码、章节顺序、译文是否缺失。
2. Pre-translation parse review：说话人归属、歧义判断是否合理。
3. Accuracy audit：说话人、指代、时间线、动作、因果。
4. Literary edit：中文-only 通读，去翻译腔（旁白/内心/对话三轨分别编辑）。
5. Voice pass：角色声音、称呼、关系距离（对照声线卡）。
6. Consistency pass：跨章术语、人设、伏笔、反复句。
7. Golden set regression：对照样张检查是否退化。
