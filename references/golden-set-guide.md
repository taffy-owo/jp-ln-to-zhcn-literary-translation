# Golden Set 建立与使用指南

## 什么是 Golden Set

Golden set 是一组**经过人工审校批准的"标杆译文段落"**。它的作用不是追求"完美翻译"，而是建立一个**可回归的基准**：每次更新 prompt、agent 或工作流后，用同样的原文跑一遍，对比新译文与 golden set 的偏离程度，从而量化"是进步了还是退步了"。

## 场景类型

Golden set 应覆盖至少 5-10 段典型场景，每种场景选 1-2 个代表性段落：

| 场景类型 | 为什么要覆盖 | 选段标准 |
|---|---|---|
| 日常甜蜜 | 轻小说最常见；最容易翻得"对但无味" | 有撒娇、小吐槽、称呼变化的互动 |
| 告白 / 感情突破 | 情感密度高，对节奏和停顿要求极高 | 有犹豫、鼓起勇气、话说到一半的段落 |
| 争吵 / 冲突 | 角色声音差异的极端测试 | 有攻击性台词、情绪升级、沉默 |
| 吐槽 / 喜剧 | 笑点是否还活着 | 有梗、punchline、对话节奏 |
| 内心创伤 / 回忆 | 压抑、痛苦、停顿的处理 | 有省略号、未说完的话、身体感 |
| 世界观说明 / 设定 | 信息密度高，容易变成说明书 | 有角色边解释边互动的段落 |
| 亲密 / 床戏 | 忠实呈现的底线测试 | 有感官细节、身体描写（无遮掩原则） |
| 配角互动 | 次要角色声线是否被主角声线同化 | 有 2+ 配角对话的段落 |

## 样张格式

每个样张一个文件，存放在 `golden_set/` 目录下：

```markdown
# Golden Sample: [编号]-[场景类型简述]

## 元信息
- 来源：[卷名 / 章节号]
- 场景类型：[从上表选择]
- 出场角色：[列出]
- 审校者：[人名或 agent 版本]
- 批准日期：[YYYY-MM-DD]
- 版本：v1

## 日文原文
> [粘贴日文原文段落]

## 批准中文译文
> [粘贴经审校批准的中文译文]

## 审校备注
- 说话人归属：[是否有歧义？如何判断？]
- 称呼选择：[为什么用这个称呼？]
- 声线要点：[哪些台词特别能体现角色声音？]
- 文化负载词处理：[如有]
- 亲密/敏感内容处理：[如有]
- 节奏与停顿：[哪些地方的留白/省略号是有意为之？]
- 已知权衡：[为了中文自然做了哪些妥协？]

## 回归检查点
翻译此段落时，以下方面不得退化：
- [ ] [具体检查项 1]
- [ ] [具体检查项 2]
- [ ] [具体检查项 3]
```

## 如何建立首批 Golden Set

### 步骤 1：选段
- 从已完成的翻译项目中选取覆盖上述场景类型的段落
- 每段 200-800 字（日文），太长不适合精确对比
- 优先选择翻译过程中反复修改过的段落（说明有难度）

### 步骤 2：审校
- 对选取的段落做最终人工审校
- 确认说话人归属、称呼、声线、节奏都已达到理想状态
- 标注已知权衡（没有完美翻译，只有当前最佳判断）

### 步骤 3：写入
- 按上述格式写入 `golden_set/` 目录
- 每个样张命名为 `gs-001-daily-sweet.md`、`gs-002-confession.md` 等

### 步骤 4：验证
- 用当前 agent/prompt 重新翻译同一段日文
- 对比新译文与批准译文的差异
- 记录偏离点：事实偏差、称呼偏差、声线偏差、节奏偏差

## 回归测试流程

每次更新 agent、prompt 或工作流后：

1. **重新翻译**：用更新后的系统翻译所有 golden set 的日文原文
2. **逐项对比**：对照批准译文，标出差异
3. **分类差异**：
   - 🟢 **进步**：新译文确实更好（更自然、更准确、声线更稳）
   - 🟡 **等价**：不同但同等质量
   - 🔴 **退化**：新译文在某个检查点上退步了
4. **决策**：
   - 如果退化 ≤ 1 项且无 critical，可以接受更新
   - 如果退化 ≥ 2 项或有 critical，需要修复后重新回归
   - 如果某个样张的新译文确实更好，更新 golden set 到新版本

## 版本管理

- golden set 样张有版本号（v1, v2, ...）
- 每次更新样张时记录原因
- 旧版本不删除，标记为 `superseded`
- 回归测试始终使用最新版本

## 增长节奏

- **首批**：5 段，覆盖最核心的场景类型
- **每完成一个翻译项目**：新增 1-2 段（如果发现了新的高难度场景类型）
- **目标**：长期维持 10-15 段，不追求数量，追求覆盖面和代表性
- **不要一次做大**：每周最多新增 1 段，保持每段的审校质量
