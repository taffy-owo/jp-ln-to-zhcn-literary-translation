# 角色声线卡模板

## 用途

轻小说最大的阅读快感之一是"一看台词就知道谁在说话"。声线卡的目的是把这种直觉感知工程化，防止在批处理、多轮编辑、跨章节翻译中声线慢慢漂掉。

## 何时建立

- **整卷任务**：Context Pass 阶段为每个主要角色（≥3 句台词的角色）建立声线卡。
- **单章任务**：如已有 `meta/voice_cards/`，先读取；如没有，至少为本章出场的主要角色建立简版。
- **更新时机**：角色经历重大关系变化（告白、决裂、和解）后，更新声线卡并标注变化章节。

## 声线卡格式

每个角色一个文件，存放在 `meta/voice_cards/<角色名>.md`。

```markdown
# 角色声线卡：【角色名】

## 基本信息
- 日文名 / 读音：
- 年龄感：（少女 / 少年 / 青年 / 成熟 / 幼童）
- 身份：（学生 / 上班族 / 教师 / 贵族 / 等）
- 性格标签：（冷淡 / 傲娇 / 元气 / 毒舌 / 温和 / 怯弱 / 等）
- 叙述角色：（主角 / 第一人称叙述者 / 配角 / 反派 / 等）

## 称呼矩阵

| 对象 | 正常 | 亲密/撒娇 | 生气/吃醋 | 害羞/尴尬 |
|---|---|---|---|---|
| 对【A】 | | | | |
| 对【B】 | | | | |
| 对陌生人 | | | | |
| 自称 | | | | |

## 句法特征
- **默认句长**：（短句为主 / 中等 / 偏长 / 视情绪变化）
- **语气词偏好**：（常用 / 偶尔用 / 几乎不用 / 特定语气词列表）
- **回避表达**：（这个角色绝对不会说的话或用的词）
- **口语强度**：（极口语 / 适度口语 / 偏书面 / 混合）
- **典型句尾**：（断句风格、省略号使用、感叹号频率）

## 情绪失控时的语言变化
- **愤怒**：（句子变短？变粗？骂人方式？）
- **悲伤**：（声音变小？断断续续？沉默？）
- **害羞**：（结巴？岔开话题？否认？）
- **开心**：（话变多？语气词增加？身体动作增加？）
- **吃醋/占有欲**：（冷淡？攻击性？粘人？）

## 与其他角色的语气差异
- 对【A】时更……
- 对【B】时更……
- 当着外人面 vs 两人独处时的差异：

## 样例台词
列出 3-5 句能代表这个角色说话方式的典型译文：

1. 「」 —— （场景/情绪说明）
2. 「」 —— （场景/情绪说明）
3. 「」 —— （场景/情绪说明）

## 声线漂移检查点
翻译和审校时，如果出现以下情况，说明声线可能漂移了：
- [ ] 这个角色的台词读起来像另一个角色
- [ ] 称呼突然变化但没有剧情原因
- [ ] 句长/语气词密度明显偏离默认值
- [ ] 情绪表达方式与性格标签不一致
- [ ] 口语强度与之前章节差异明显

## 更新记录
| 章节 | 变化 | 原因 |
|---|---|---|
```

## 使用方式

### 在 Context Pass 中
1. 读完整卷或足够多的章节后，为主要角色填写声线卡
2. 如果有前卷已有声线卡，先读取再更新

### 在翻译中
1. 翻译台词时参照声线卡的称呼矩阵和句法特征
2. 遇到情绪变化场景时参照"情绪失控时的语言变化"
3. 不确定时宁可保守（贴近默认值），不要即兴发挥

### 在审校中
1. reviewer 按声线漂移检查点逐项检查
2. 发现漂移时标注为 `voice_drift` 类型问题
3. 重大声线变化（如关系突变后称呼改变）应更新声线卡并在 continuity_notes 中记录

## 简版声线卡

次要角色（台词 < 10 句）可以不建完整卡，但至少记录：

```markdown
## 【角色名】（简版）
- 性格：
- 对主角称呼：
- 说话特点（一句话）：
```
