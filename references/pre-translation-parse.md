# 译前歧义解析指南

## 为什么要做译前解析

日轻翻译中，大多数严重错误不是来自不认识词，而是来自：
- 省略主语导致说话人/行动者归属错误
- 内心话与说出口的话混淆
- 敬语的关系功能被扁平化
- 看似普通的台词实际带讽刺/撒娇/试探/逞强语气
- 多人对话中话轮归属不明

translator 在首译前必须先做 `pre-translation parse`，不直接翻译，只做理解与记录。

## 何时触发

- **必须做**：每章首译前。
- **可跳过**：用户明确要求"只要快速初译"时，但需在 QA 中标注"未做译前解析"。
- **局部做**：单段或单场景的补译/改稿，只解析相关段落。

## 解析模板

对每个段落或场景，输出以下结构化信息：

```markdown
### 段落 [ch0X-scYY-pZZ]

| 维度 | 判断 |
|---|---|
| 说话人 | （谁在说/谁在想） |
| 受话人 | （说给谁听） |
| 隐含主语 | （省略了谁？补出理由是什么？） |
| 敬语功能 | （礼貌/距离/上下级/年龄差/亲昵反转？） |
| 情绪/语气功能 | （安慰/挑衅/拒绝/逞强/撒娇/试探/转移话题/自嘲？） |
| 信息焦点 | （这句话要传达的核心信息是什么？） |
| 歧义点 | （有没有多种合理理解？哪种最可能？） |
| 风险等级 | low / medium / high |
```

## 高风险句型清单

以下情况风险等级默认为 **high**，必须在解析中明确判断：

### 1. 省略主语的多人对话
日文允许在多人在场时连续省略主语。如果不先确定话轮归属，译文很可能把 A 说的话写成 B 说的。

**判断方法**：
- 看前后文的敬语/粗语切换
- 看语气词（わ/よ/ぜ/かな/だろう）与角色匹配
- 看动作描写穿插的位置
- 看回应内容是否逻辑上对应前一句

### 2. 内心话 vs 说出口的话
日文用「」和内心叙述交替出现。有时一段既有说出口的台词，又有内心吐槽。

**判断方法**：
- 「」内是说出口的
- 心の中で / ～と思った / 内心 / 胸の奥 等标记内心
- 没有明确标记时，看这句话的攻击性/坦率度是否超出角色对外表现

### 3. 敬语层级切换
同一角色在不同对象面前切换敬语，编码的是关系距离和权力结构。

**判断方法**：
- です/ます → 礼貌/距离/初识/正式
- 突然从敬语切到非敬语 → 亲近/失控/愤怒/试探
- 对方切换敬语 → 关系变化信号，必须记录到 continuity_notes

### 4. 讽刺 / 撒娇 / 试探 / 逞强
字面意思和真实意图相反的台词。如果只翻字面，人物会变平。

**判断方法**：
- 看角色性格标签（傲娇/毒舌/怯弱）
- 看上下文反应（对方是否被逗笑/被刺痛/被感动）
- 看是否有"～のくせに""べつに""～なんて"等反转标记

### 5. 暧昧与未说完
日文大量使用"……"和句尾省略来表达犹豫、暗示、不敢说出口的话。

**判断方法**：
- 省略号后对方的反应暗示了未说完的内容
- 角色性格决定了省略的含义（害羞 vs 故意留白 vs 威胁）

## 歧义登记

风险等级为 high 的歧义点，如果无法在当前上下文中确定唯一解释，必须写入 `meta/open_issues.md`：

```markdown
| ID | Severity | Location | Issue | Current decision | Needs review |
|---|---|---|---|---|---|
| AMB-043-01 | high | ch043 p12 | 「……そうだね」的说话人不确定，可能是怜也可能是爱 | 暂定怜（基于前一句动作描写） | 是 |
```

## Prompt 示例

```text
请只做译前解析，不要翻译。
对以下日文段落，逐段输出：
- 说话人与受话人
- 隐含主语及补出理由
- 敬语功能（礼貌/距离/亲昵/愤怒？）
- 情绪/语气功能（安慰/试探/撒娇/逞强/自嘲？）
- 信息焦点
- 歧义点与最可能解释
- 风险等级（low/medium/high）

对 high 风险的歧义点，给出两种可能解释及各自依据。
```

## 与翻译流程的衔接

1. 完成 parse log 后，将 high 风险项记录到 `meta/open_issues.md`
2. translator 翻译时必须参照 parse log，不得无视歧义判断直接按直觉译
3. editor 编辑时检查 parse log 中的说话人归属是否在译文中正确体现
4. reviewer 审校时将 parse log 作为证据之一，检查"是否翻对了意思"
